# 示例 10：循环依赖

本示例展示了本库原生支持属性注入循环依赖的能力，以及为什么能做到这一点。

## 展示的特性

- **简单循环依赖 A ↔ B**：两个服务互相依赖，本库可以正常解析
- **三角循环依赖 A → B → C → A**：多个服务形成环形依赖链，同样支持
- **LazyToken 解决前向引用**：两个类互相引用时，使用 `new LazyToken(() => ClassName)` 解决 JS 模块加载顺序问题
- **@PostConstruct 中访问循环依赖**：`@PostConstruct` 在属性注入之后执行，可以安全访问循环依赖的属性

## 为什么能支持循环依赖？

本库的解析流程：
1. `new ClassName()` 创建实例
2. 执行激活处理器
3. **将实例存入缓存**（关键步骤）
4. 注入属性依赖

由于实例在属性注入之前就已缓存，当解析循环依赖时可以直接从缓存取到已创建的实例，不会无限递归。

> inversify 默认使用构造函数参数注入，实例创建前无法缓存，因此需要 `LazyServiceIdentifier` 才能支持循环依赖。
