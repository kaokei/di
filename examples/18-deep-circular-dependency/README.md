# 示例 18：深层循环依赖

本示例通过一个 8 个节点的深层循环依赖链（A→B→C→D→E→F→G→H→C），验证本库对复杂循环依赖的支持能力。

## 展示的特性

- **深层循环依赖链**：`A→B→C→D→E→F→G→H→C`，其中 H 又依赖回 C，形成嵌套循环
- **属性注入天然支持循环依赖**：无论依赖链多深，属性注入都不会出现无限递归
- **LazyToken 解决前向引用**：在同一文件中互相引用的类，使用 `new LazyToken(() => ClassName)` 解决声明顺序问题

## 背景说明

旧版本（基于 `experimentalDecorators`）支持构造函数参数注入，构造函数注入在循环依赖时会无限递归，需要框架主动检测并抛出 `CircularDependencyError`。

当前版本基于 TC39 Stage 3 装饰器规范，只支持属性注入。属性注入的机制是先创建实例并缓存，再注入属性，因此天然不存在循环依赖问题。
